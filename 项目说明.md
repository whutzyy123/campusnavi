# 校园生存指北 - 项目说明文档

## 📋 项目概述

**校园生存指北 (Campus Survival Guide)** 是一个精细化校区地理信息系统（GIS），致力于实现校区内导航"最后一米"的精准交付。

### 项目背景

传统地图服务（如高德、百度）对校区内部覆盖粒度不足，缺乏校内小路、建筑出入口等精细化数据。同时，校内临时封路、施工信息、食堂排队状态等实时信息更新滞后，无法满足校园用户的精准导航需求。

### 核心价值

1. **解决"最后一米"导航问题**：通过众包数据和高德地图步行导航，实现从起点到具体教室、建筑入口的精准导航
2. **校园社交与信息聚合**：POI 留言板功能，让用户分享实时状态、交流信息，形成校园内的信息聚合平台
3. **多租户架构**：支持多所学校独立运营，数据完全隔离，可扩展性强

### 项目目标

- **业务目标**：打造国内领先的精细化校区地理信息系统，实现校区内导航"最后一米"的精准交付
- **用户价值**：提供基于众包的实时校内路况、精准建筑入口导航及校内资源分布图
- **核心指标**：导航任务完成率、POI 状态更新频次、多租户（学校）覆盖数、用户纠错采纳率

### 多租户架构设计

系统采用基于 `schoolId` 的数据隔离策略：

1. **数据隔离**：所有业务表（POI、Comment、LiveStatus 等）必须包含 `schoolId` 字段
2. **查询过滤**：所有数据查询必须包含 `where: { schoolId: currentSchoolId }` 条件
3. **权限隔离**：用户只能访问其所属学校的数据（超级管理员除外）
4. **学校绑定**：
   - **已绑定用户**（Student、Admin、Staff）：强制锁定 `currentUser.schoolId`，无法切换
   - **未绑定用户**（Guest、Super Admin）：可通过 Navbar 下拉选择器自由切换学校

## 🧩 核心功能模块与实现逻辑

### A. 地图与学校加载逻辑（重构版）

#### 业务流程

系统根据用户角色和学校绑定关系，采用不同的加载策略：

1. **已绑定学校的用户**（Student、Admin、Staff）
   - 页面加载时，直接读取 `currentUser.schoolId`
   - 强制将地图中心设置为该学校的中心坐标（`school.center`）
   - 忽略用户的实时 GPS 坐标（仅用于显示蓝点，不自动平移地图）
   - 直接加载该学校的 POI 数据

2. **未绑定学校的用户**（Guest、Super Admin）
   - 页面加载时，默认不锁定特定学校
   - 地图中心以用户的实时 GPS 坐标为初始中心
   - 在 Navbar 提供"切换学校"下拉选择框，允许手动选择学校
   - 选择学校后，地图视角平移到该学校，并加载该学校数据

#### 关键技术实现

**代码位置**: `app/page.tsx`

```typescript
// 根据用户 schoolId 绑定强制加载学校
useEffect(() => {
  const loadLockedSchool = async () => {
    if (!currentUser?.schoolId) return;
    
    const response = await fetch(`/api/schools/${currentUser.schoolId}`);
    const data = await response.json();
    if (data.success && data.school) {
      setActiveSchool(data.school);
    }
  };
  
  loadLockedSchool();
}, [currentUser?.schoolId]);
```

#### LOD (Level of Detail) 优化

**功能描述**：地图缩小时自动隐藏 POI 标记，减少视觉干扰

**实现逻辑**：

1. **缩放级别监听**：在 `components/poi-map.tsx` 中监听 `map.on('zoomchange')` 事件
2. **动态属性**：根据当前缩放级别，给地图容器添加 `data-zoom-level` 属性（`"low"` 或 `"high"`）
3. **CSS 控制**：通过 CSS 选择器 `.hide-pois .amap-marker` 控制 POI 标记的可见性
4. **阈值设定**：当 `zoom < 16` 时，隐藏所有 POI 标记；`zoom >= 16` 时，显示 POI 标记

**代码位置**: `components/poi-map.tsx`

```typescript
// 根据缩放级别更新容器属性
const updateZoomLevelAttr = () => {
  const currentZoom = mapInstanceRef.current.getZoom();
  const level = currentZoom < 16 ? "low" : "high";
  mapRef.current.setAttribute("data-zoom-level", level);
};

map.on("zoomend", updateZoomLevelAttr);
map.on("zoomchange", updateZoomLevelAttr);
```

**性能优化**：
- 使用 CSS 类名控制，避免频繁重新创建 Marker
- 利用 `transform: scale()` 实现平滑缩放动画
- 不影响用户定位蓝点的显示

### B. 校内路径规划与导航（新增）

#### 功能概述

基于高德地图 `AMap.Walking` 插件，实现校内步行路径规划和实时导航指引。

#### 技术实现

**状态管理**: `store/use-navigation-store.ts`

- **状态字段**：
  - `startPoint` / `endPoint`: 起点和终点坐标
  - `isNavigating`: 是否处于导航模式
  - `routeInfo`: 路径信息（距离、时间、路径点）
  - `routeSteps`: 导航步骤（"向东步行 100 米"等）
  - `selectMode`: 当前选点模式（`"start"` | `"end"` | `null`）

- **核心 Actions**：
  - `setStartPoint` / `setEndPoint`: 设置起终点
  - `setSelectMode`: 进入地图选点模式
  - `startNavigation`: 开始导航
  - `clearNavigation`: 清除导航状态

**地图集成**: `components/poi-map.tsx`

1. **插件预加载**：在 `lib/amap-loader.ts` 中统一加载 `AMap.Walking` 插件
2. **路径计算**：当 `isNavigating` 为 `true` 且起终点都存在时，调用 `AMap.Walking.search()` 进行路径规划
3. **路径绘制**：使用 `AMap.Polyline` 在地图上绘制导航路线（蓝色高亮）
4. **地图选点**：监听地图点击事件，当 `selectMode` 不为空时，捕获坐标并更新 Store

**交互流程**：

1. **发起导航**：用户在 POI 详情页点击"到这去"按钮
   - 自动将当前 POI 设为 `endPoint`
   - 尝试获取用户当前位置设为 `startPoint`（如果获取不到，留空让用户选）
   - 打开导航面板，进入导航模式

2. **选点方式**：
   - **POI 搜索**：在导航面板输入框搜索 POI，选中后自动设为起点/终点
   - **地图选点**：点击"地图选点"按钮，进入选点模式，在地图空白处点击即可选点

3. **路径展示**：
   - 地图上绘制蓝色导航路线
   - 导航面板显示预估距离（米）和步行时间（分钟）
   - 展开"路线详情"折叠面板，显示具体行走步骤

**代码位置**: `components/navigation-panel.tsx`, `components/poi-map.tsx`

### C. POI 社交与审核系统（新增）

#### 功能概述

为每个 POI 提供留言板功能，支持用户发布留言、敏感词过滤、举报机制，以及管理员审核流程。

#### 数据库设计

**模型**: `prisma/schema.prisma` - `Comment` 表

```prisma
model Comment {
  id          String   @id @default(cuid())
  content     String   @db.Text
  userId      String   @map("user_id")
  poiId       String   @map("poi_id")
  schoolId    String   @map("school_id")
  isHidden    Boolean  @default(false)  // 是否被隐藏/屏蔽
  reportCount Int      @default(0)       // 被举报次数
  createdAt   DateTime @default(now())
  
  user        User     @relation(...)
  poi         POI      @relation(...)
  school      School   @relation(...)
}
```

#### 业务流程

**用户侧**（`components/poi-drawer.tsx`）：

1. **留言发布**：
   - 在 POI 详情抽屉底部显示留言板区域
   - 用户输入留言内容，点击"发送"
   - 后端进行敏感词过滤（`lib/content-validator.ts`）
   - 如果包含敏感词，直接拒绝（返回 400）或自动标记为 `isHidden: true`

2. **留言列表**：
   - 显示留言列表（包含头像、昵称、时间、内容）
   - 默认只返回 `isHidden: false` 的留言
   - 支持分页加载（`page`, `limit` 参数）

3. **举报与删除**：
   - 每条留言提供"举报"按钮（小图标）
   - 如果是当前用户自己的留言，显示"删除"按钮
   - 举报后，对应留言的 `reportCount +1`
   - 如果 `reportCount >= 5`，自动将 `isHidden` 设为 `true`

**管理员侧**（`app/admin/audit/comments/page.tsx`）：

1. **审核列表**：
   - 获取本校所有 `reportCount > 0` 或 `isHidden = true` 的留言
   - 显示：留言内容、作者、所属 POI、被举报次数、发布时间

2. **审核操作**：
   - **通过/恢复**：将 `isHidden` 设为 `false`，清空 `reportCount`
   - **删除**：物理删除该留言

#### API 接口

- `GET /api/comments?poiId=xxx&page=1&limit=20` - 获取留言列表
- `POST /api/comments` - 创建留言（敏感词过滤）
- `DELETE /api/comments/[id]` - 删除留言（权限：作者本人、校管理员、超级管理员）
- `POST /api/comments/[id]/report` - 举报留言
- `GET /api/admin/comments` - 获取待审核留言（管理员）
- `POST /api/admin/comments/[id]/restore` - 恢复留言（管理员）

**代码位置**: `app/api/comments/`, `components/poi-drawer.tsx`

### D. 用户与权限管理

#### RBAC 模型

系统采用 5 级权限体系：

| 角色 | Role Number | 权限说明 |
|------|-------------|----------|
| **游客** | 0 | 只读地图，无法发布内容 |
| **认证学生** | 1 | 可发布众包信息、参与导航、留言 |
| **校级管理员** | 2 | POI 管理、团队管理、举报审核、留言审核 |
| **校内工作人员** | 3 | 举报审核、留言审核 |
| **超级管理员** | 4 | 系统管理、用户管理、敏感词管理、跨学校访问 |

#### 认证安全

**技术实现**: `lib/auth-server-actions.ts`, `components/auth-guard.tsx`

1. **HTTP Only Cookie**：
   - 所有认证信息存储在服务端设置的 HTTP Only Cookie 中
   - 防止 XSS 攻击，前端 JavaScript 无法访问 Cookie

2. **Server Actions**：
   - 所有认证操作（登录、注册、登出）使用 Next.js Server Actions
   - 密码哈希使用 SHA-256（生产环境建议使用 bcrypt）

3. **权限守卫**：
   - `AuthGuard` 组件进行客户端权限验证
   - 支持 `requiredRole` 和 `requireSchoolId` 参数
   - **Hydration 处理**：在权限检查期间显示 Loading 状态，避免页面闪烁

**代码位置**: `components/auth-guard.tsx`

```typescript
// 在权限检查期间显示加载状态，避免页面看起来无响应
```

   - 学校数量统计
   - 今日新增用户数

**代码位置**: `app/super-admin/users/page.tsx`, `app/api/admin/users/route.ts`

---

### E. 基础设施

#### 高德插件预加载

**技术实现**: `lib/amap-loader.ts`

1. **统一加载**：在应用初始化时，使用 `AMapLoader.load()` 一次性加载所有必要插件：
   - `AMap.Walking` - 步行导航
   - `AMap.Polyline` - 折线绘制
   - `AMap.Marker` - 点标记
   - `AMap.Polygon` - 多边形（用于画边界）
   - `AMap.PlaceSearch` - 地点搜索
   - `AMap.AutoComplete` - 自动完成

2. **安全密钥配置**：
   - 支持 `NEXT_PUBLIC_AMAP_SECURITY_KEY` 环境变量
   - 通过 `window._AMapSecurityConfig` 配置安全密钥

3. **坐标验证**：
   - `CoordinateConverter.isValidGCJ02()` 验证坐标是否在 GCJ-02 服务范围内
   - 防止导航服务返回 `OUT_OF_SERVICE` 错误

**代码位置**: `lib/amap-loader.ts`

#### Server Actions 错误处理

**技术实现**: `lib/auth-server-actions.ts`

1. **NEXT_REDIRECT 处理**：
   - Next.js 的 `redirect()` 函数通过抛出异常实现跳转
   - 在 `catch` 块中显式检查 `error.message.includes("NEXT_REDIRECT")`
   - 如果是跳转信号，直接返回，不显示错误信息

2. **错误提示**：
   - 使用 `react-hot-toast` 显示友好的错误提示
   - 区分业务错误和系统错误

**代码位置**: `app/register/page.tsx`, `lib/auth-server-actions.ts`

## 📝 开发规范

### 代码规范

- 使用 TypeScript Strict 模式
- 遵循 Next.js App Router 最佳实践
- 组件采用原子化设计
- Server Components 优先，Client Components 按需使用（`"use client"`）

### 数据库规范

- 所有业务表必须包含 `schoolId` 字段
- 所有查询必须包含 `where: { schoolId: currentSchoolId }`（超级管理员除外）
- 使用 Prisma 进行类型安全的数据库操作
- 所有时间字段使用 `DateTime` 类型，自动处理时区

### 认证规范

- 所有认证操作使用 Server Actions
- 认证状态存储在 HTTP Only Cookie 中
- 前端不持久化认证状态，通过 `/api/auth/me` 获取
- `AuthGuard` 组件必须处理 Hydration 状态，避免页面闪烁

### 地图规范

- 所有坐标使用 GCJ-02 坐标系
- 使用 `CoordinateConverter` 验证坐标有效性
- 高德地图插件统一在 `lib/amap-loader.ts` 中预加载
- 地图组件必须处理加载失败和错误状态
