# 数据库设计文档

本文档描述 **Campus Survival Guide（校园生存指北）** 的数据库模型，唯一直据来源为仓库根目录 **`prisma/schema.prisma`**。若代码或迁移与本文不一致，以 **Prisma schema 与实际迁移**为准。

---

## 1. 概要

| 项目 | 说明 |
|------|------|
| 数据库引擎 | MySQL |
| ORM | Prisma 5.x |
| 连接配置 | `DATABASE_URL`（环境变量） |
| 主键策略 | `cuid()`（字符串）；集市部分表使用整型自增 |
| 多租户 | **绝大多数业务实体按 `schoolId` 归属学校**；下文注明例外 |

### 1.1 租户与边界约定

- **School**：租户根实体；校内地图 POI、路径、活动等均挂在 `schoolId` 下。
- **CampusArea**：在同一 `School` 下支持**多校区**，地理边界由 `boundary`（GeoJSON）、`center`、`labelCenter` 描述。
- **写入侧**：服务端必须从会话 / Cookie 注入 `schoolId`，**不信任**客户端伪造的租户字段（与 `docs/开发规范.md` 一致）。
- **例外（无 schoolId）**：
  - **Feedback**：按用户归属，表中**无** `school_id`（见下文 Feedback 小节）。
  - **Global Category**：`Category.schoolId === null` 时表示全局分类；另有 `CategoryOverride` 做各校覆盖。
  - **超级管理员**：`User.role = 4` 时 **`schoolId` 应为 null**，与单一学校解耦。

---

## 2. 用户角色（`users.role` 整型）

| 数值 | 语义（注释约定） |
|------|------------------|
| 0 | 游客 |
| 1 | 认证学生 |
| 2 | 校级管理员 |
| 3 | 校内工作人员 |
| 4 | 超级管理员 |

业务代码须使用 **统一映射函数**处理角色，禁止未知值静默降级为某一角色（参见开发规范）。

---

## 3. 实体关系总览（按领域）

```
School ─┬─ CampusArea / POI / RouteEdge / LiveStatus / Activity / LostFoundEvent
        ├─ Category (+ CategoryOverride) / Comment / InvitationCode / MarketItem ...
        └─ User（含 AuthSession、各类 UGC、集市行为）

MarketTransactionType ─── MarketTypeCategory ─── MarketCategory
MarketItem ─── MarketIntention / MarketLog
POI ─── children(self) / favorites / comments / statuses / ...
User ─── Notification（收发双方）
```

以下为各表字段级说明（仅列核心字段与设计意图；完整列以 schema 为准）。

---

## 4. 学校与地理

### 4.1 `schools` — School

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String(cuid) | PK |
| name | VarChar(100) | 学校名称 |
| school_code | VarChar(50) | **唯一**，如 `pku` |
| center_lat / center_lng | Float? | 无校区数据时的地图回退中心 |
| is_active | Boolean | 软删除开关 |

索引：`@@index([centerLat, centerLng])`。

### 4.2 `campus_areas` — CampusArea

| 字段 | 说明 |
|------|------|
| school_id | FK → School |
| name | 校区名称 |
| boundary | Json，GeoJSON 多边形 |
| center | Json，`[lng, lat]` |
| label_center | Json?，标签点（如 polylabel） |

---

## 5. 用户与会话

### 5.1 `users` — User

| 字段 | 说明 |
|------|------|
| school_id | FK?；超管应为 null |
| phone / email | 可选，unique |
| password | 哈希 |
| role | Int，见 §2 |
| points | 积分 |
| status |默认 `ACTIVE`（`INACTIVE` 表示停用等）|
| last_profile_update_at | 昵称/头像等上次修改（用于冷却等业务）|

### 5.2 `auth_sessions` — AuthSession

会话令牌、过期时间、可按需吊销（`revoked_at`）。

### 5.3 `rate_limits` — RateLimit

通用限流：`key` + `count` + 时间窗口 `window_start`。

---

## 6. 邀请码

### 6.1 枚举 `InvitationCodeType`

- `ADMIN` — 校级管理员邀请  
- `STAFF` — 校内工作人员邀请  

### 6.2 枚举 `InvitationCodeStatus`

- `ACTIVE` — 可用  
- `USED` — 已绑定用户消耗  
- `DISABLED` — 已撤销  
- `DEACTIVATED` — 停用（可与禁止登录等业务联动，以应用逻辑为准）

### 6.3 `invitation_codes` — InvitationCode

| 要点 | 说明 |
|------|------|
| code | unique |
| used_by_user_id | unique；每用户至多消耗一条邀请链 |
| expires_at | 可选过期时间 |

---

## 7. POI 分类体系

### 7.1 `categories` — Category

| 字段 | 说明 |
|------|------|
| school_id | null = 全局分类 |
| is_global | 是否为全局分类 |
| is_micro_category | 「便民设施」类微观分类（通常超管维护）|

约束：`@@unique([schoolId, name])`。

### 7.2 `category_overrides` — CategoryOverride

各校对**全局分类**的隐藏（`is_hidden`）或显示名覆盖（`custom_name`）。

约束：`@@unique([schoolId, categoryId])`。

---

## 8. POI 与校内导航

### 8.1 `pois` — POI

| 字段 | 说明 |
|------|------|
| school_id | 必填 |
| parent_id | 子 POI → 父 POI |
| category_id | FK Category；兼容字段 `category`  varchar |
| lat / lng | Decimal(10,8)，GCJ-02 |
| is_official | PGC vs UGC |
| entry_points | Json? |
| image_url | 外链，非 Base64 |
| report_count | ≥业务阈值时可自动隐藏 |

关系：`children`（自关联）、`liveStatuses`、`comments`、`activities`、`lostFoundEvents`、`marketItems`、`favorites`。

### 8.2 `poi_favorites` — POIFavorite

约束：`@@unique([userId, poiId])`；冗余 `schoolId` 便于按校查询。

### 8.3 `route_edges` — RouteEdge

校内路网补边：`start_node` / `end_node` Json `[lng, lat]`，`weight`、`type`。

---

## 9. 实时情报（TTL）

### `live_statuses` — LiveStatus

| 字段 | 说明 |
|------|------|
| poi_id / school_id | FK |
| user_id | 可选匿名 |
| status_type | VarChar，如拥挤/施工/关闭等 |
| expires_at | **TTL**：人流短、事件/较长（具体时长由应用层写入）|

索引含 `[schoolId, expiresAt]` 便于清理与查询。

---

## 10. 活动与失物招领

### 10.1 `activities` — Activity

挂靠 `schoolId`、`poiId`、`created_by`，含时间窗 `start_at` ~ `end_at`。

### 10.2 `lost_found_events` — LostFoundEvent

枚举 **`LostFoundStatus`**：`ACTIVE` | `FOUND` | `EXPIRED` | `HIDDEN`。  
`images`：`Json` 数组 URL；`expires_at` 由服务端计算。

---

## 11. 通知与反馈

### 11.1 `notifications` — Notification

枚举 **`NotificationType`**：`LIKE`、`REPLY`、`MENTION`、`SYSTEM`、`LOST_FOUND_FOUND`。  
枚举 **`NotificationEntityType`**：`COMMENT`、`LOST_FOUND`、`POI`、`MARKET_ITEM`。  
`user_id` 收件人，`actor_id` 可选触发者。

### 11.2 `feedbacks` — Feedback

**无 `schoolId`**：平台级反馈，与用户绑定。

枚举 **`FeedbackType`**：`FEEDBACK`、`BUG`。  
枚举 **`FeedbackStatus`**：`PENDING`、`RESOLVED`、`REJECTED`。

---

## 12. 内容与审核（留言）

### 12.1 `sensitive_words` — SensitiveWord

全局屏蔽词：`keyword` unique；`added_by_id` 必须为超管（应用层约束）。

### 12.2 `comments` — Comment

| 字段 | 说明 |
|------|------|
| school_id / poi_id | 租户与载体 |
| parent_id | 楼中楼回复 |
| is_hidden | 屏蔽展示 |
| report_count | 举报计数 |
| like_count | **冗余**，热度排序 |
| is_reviewed / reviewed_at / reviewed_by | 管理审核 |

### 12.3 `comment_likes` — CommentLike

`@@unique([userId, commentId])`。

### 12.4 `comment_logs` — CommentLog（审计）

枚举 **`CommentLogActionType`**：当前含 `HARD_DELETED`。  
`comment_id` 在删除后可 SetNull，保留审计行。

---

## 13. 生存集市（校内二手/交易闭环）

### 13.1 配置类

| 物理表 | 模型 | 说明 |
|--------|------|------|
| market_categories | MarketCategory | 扁平物品分类 |
| market_transaction_types | MarketTransactionType | 动态交易类型（含 `code`）|
| market_type_categories | MarketTypeCategory | **复合 PK** `(transactionTypeId, categoryId)` 多对多 |

### 13.2 枚举 `MarketItemStatus`

| 值 | 含义 |
|----|------|
| ACTIVE | 在架 |
| LOCKED | 意向锁定，待交割 |
| HIDDEN | 管理员下架 |
| COMPLETED | 已完成 |
| DELETED | 用户删除 |

约定（注释）：典型流转 **ACTIVE → LOCKED → COMPLETED / DELETED**；`HIDDEN` 可通过管理操作回到 `ACTIVE`。

### 13.3 `market_items` — MarketItem

核心字段：`school_id`、`poi_id`、`user_id`、`type_id`、`category_id?`、`price?`、`images` Json、`status`、`expires_at`（如创建 +7 天）。

闭环字段：`selected_buyer_id`、`buyer_confirmed`、`seller_confirmed`、`locked_at`、`first_confirmed_at`、`buyer_rating_of_seller`、`seller_rating_of_buyer`。

### 13.4 `market_intentions` — MarketIntention

同一用户对同一商品仅一条：`@@unique([itemId, userId])`。

### 13.5 `market_logs` — MarketLog

枚举 **`MarketLogActionType`**：意向、锁定/解锁、双方确认、超时自动、管理下架/重新上架、删除等全生命周期审计。  
`item_id` 可 SetNull 保留日志。

---

## 14. 索引与性能提示（摘要）

- **多租户列表**：大量表在 `school_id` 单列或组合上建索引。  
- **POI 空间/列表**：`lat, lng`、`parentId`、`categoryId`。  
- **TTL 清理**：`live_statuses`、`market_items` 等含 `expires_at` 索引。  
- **留言热度**：`comments.like_count`、`created_at`。  
- **邀请码/限流**：按 `code`、`status`、`windowStart` 等查询场景建索引。

---

## 15. 级联删除（设计层摘要）

Prisma 关系中广泛使用 **`onDelete: Cascade`**（学校删除将级联其子实体）与 **`SetNull`**（保留审计或弱关联）。  

**注意**：`MarketItem` → `MarketTransactionType` 为 **`Restrict`**，防止误删仍被引用的交易类型。
