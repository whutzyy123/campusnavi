## 一、概述

### 1.1 文档目的

本文档定义校园生存指北产品的数据埋点规范，用于：

- **产品迭代**：了解用户行为与功能使用情况
- **运营分析**：核心指标监控、漏斗转化、留存分析
- **问题排查**：异常路径、错误率、性能监控
- **未来扩展**：A/B 测试、个性化推荐、智能运营

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **隐私优先** | 不采集敏感信息（密码、完整联系方式）；用户 ID 可脱敏或哈希 |
| **合规** | 符合《个人信息保护法》等法规；提供隐私政策与用户同意机制 |
| **平台无关** | 事件定义与上报格式与具体平台（GA4、Mixpanel、自建）解耦 |
| **可扩展** | 预留扩展字段；支持未来新增模块与事件 |
| **一致性** | 统一命名、统一属性结构，便于跨端与跨版本分析 |

### 1.3 适用范围

- Web App（主站、管理后台、超管后台）
- 未来：小程序、App（若扩展）

## 二、事件命名规范

### 2.1 命名格式

```
{模块}_{动作}_{对象}
```

- **模块**：小写英文，如 `map`、`poi`、`market`、`auth`
- **动作**：动词，如 `view`、`click`、`submit`、`search`
- **对象**：可选，如 `poi`、`comment`、`item`

### 2.2 动作词表

| 动作 | 含义 | 示例 |
|------|------|------|
| view | 页面/模块进入 | page_view |
| click | 点击（无业务结果） | button_click |
| search | 发起搜索 | poi_search |
| submit | 提交表单/操作 | comment_submit |
| success | 操作成功 | login_success |
| fail | 操作失败 | login_fail |
| select | 选择/切换 | school_select |
| open | 打开弹窗/抽屉 | drawer_open |
| close | 关闭弹窗/抽屉 | drawer_close |
| share | 分享 | item_share |
| report | 举报 | comment_report |

### 2.3 通用属性前缀

| 前缀 | 含义 | 示例 |
|------|------|------|
| `page_` | 页面相关 | page_path, page_title |
| `user_` | 用户相关 | user_id_hash, user_role |
| `school_` | 学校相关 | school_id, school_code |
| `poi_` | POI 相关 | poi_id, poi_name |
| `item_` | 集市商品相关 | item_id, item_type |
| `duration_` | 时长（毫秒） | duration_ms |
| `error_` | 错误相关 | error_code, error_message |

## 五、实现指南

### 5.1 埋点封装层

建议在 `lib/analytics.ts` 或 `lib/track.ts` 中封装统一上报接口：

```typescript
// 示例：平台无关的埋点接口
interface TrackEvent {
  name: string;
  properties?: Record<string, string | number | boolean | null>;
}

export function track(event: TrackEvent): void {
  // 1. 注入通用上下文
  // 2. 根据环境变量决定是否上报、上报到哪个平台
  // 3. 可对接 GA4、Mixpanel、自建等
  if (typeof window !== "undefined" && process.env.NEXT_PUBLIC_ANALYTICS_ENABLED === "true") {
    // 实际调用：gtag、mixpanel.track、fetch 等
  }
}
```

### 5.2 调用示例

```typescript
// 页面浏览
track({ name: "page_view", properties: { page_path: "/", page_title: "首页" } });

// POI 抽屉打开
track({ name: "poi_drawer_open", properties: { poi_id: "xxx", source: "marker" } });

// 登录成功
track({ name: "auth_login_success", properties: { user_role: "STUDENT" } });
```

### 5.3 集成时机

| 阶段 | 建议 |
|------|------|
| 开发期 | 先实现封装层与核心事件（page_view、auth、poi、market） |
| 上线前 | 配置生产环境平台（GA4 等），开启采样或全量 |
| 迭代期 | 按需求逐步补充管理端、超管端事件 |

### 5.4 环境变量

| 变量 | 说明 |
|------|------|
| NEXT_PUBLIC_ANALYTICS_ENABLED | 是否启用埋点（true/false） |
| NEXT_PUBLIC_GA_MEASUREMENT_ID | GA4 Measurement ID（若使用） |
| NEXT_PUBLIC_MIXPANEL_TOKEN | Mixpanel Token（若使用） |

## 七、隐私与合规

### 7.1 不采集内容

- 密码、完整手机号、完整邮箱
- 留言/商品描述等 UGC 正文（仅可采集 ID、类型等元数据）
- 精确 GPS 坐标（若需可模糊化）

### 7.2 用户同意

- 首次访问或关键操作前，展示隐私政策与埋点说明
- 提供「不同意」选项，不同意时不上报或仅上报匿名聚合
- 用户 ID 采用哈希或匿名 ID，避免直接暴露

### 7.3 数据保留

- 按平台与法规要求设置保留周期
- 支持用户请求删除个人相关埋点数据（若平台支持）
