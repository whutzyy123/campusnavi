# 管理员校区初始化功能使用说明

## 功能概述

管理员校区初始化功能允许管理员在地图上手动勾画学校的地理边界（多边形），并保存到数据库中。这是多租户架构的第一步。

## 访问路径

访问 `/admin/school/init` 页面即可使用该功能。

## 使用步骤

### 1. 打开页面
在浏览器中访问：`http://localhost:3000/admin/school/init`

### 2. 填写学校信息
在右侧侧边栏中填写：
- **学校名称**：例如"北京大学"
- **唯一代码**：例如"pku"（仅支持小写字母和数字）

### 3. 勾画边界
- 在地图上点击任意位置，会在点击处添加一个红色标记点
- 继续点击其他位置，系统会自动将标记点连接成多边形
- 至少需要 3 个点才能构成有效的多边形边界

### 4. 清除边界（可选）
- 如果对当前边界不满意，点击"清除边界"按钮重置
- 清除后可以重新勾画

### 5. 保存并初始化
- 确认边界和学校信息无误后，点击"保存并初始化"按钮
- 系统会：
  - 验证学校代码是否唯一
  - 计算多边形的中心点（使用 Turf.js）
  - 将边界数据存储为 GeoJSON 格式
  - 保存到数据库

## 技术实现

### 前端
- 使用高德地图 JS SDK 2.0 显示地图
- 点击地图添加标记点（Marker）
- 自动连接标记点形成多边形（Polygon）
- 实时显示已添加的边界点数量

### 后端
- API 路由：`POST /api/admin/school`
- 使用 Turf.js 的 `centroid` 函数计算多边形中心点
- 边界数据以 GeoJSON 格式存储到数据库
- 学校代码唯一性验证

### 数据格式

**边界坐标格式**：
```json
[
  [lng1, lat1],
  [lng2, lat2],
  [lng3, lat3],
  ...
]
```

**存储到数据库的格式**（GeoJSON）：
```json
{
  "type": "Polygon",
  "coordinates": [[
    [lng1, lat1],
    [lng2, lat2],
    [lng3, lat3],
    [lng1, lat1]  // 闭合：首尾相连
  ]]
}
```

## 注意事项

1. **坐标系**：所有坐标使用 GCJ-02 坐标系（高德地图标准）
2. **最小点数**：至少需要 3 个点才能构成有效的多边形
3. **学校代码唯一性**：每个学校代码必须唯一，不能重复
4. **坐标验证**：系统会自动验证坐标是否在有效范围内（中国境内）

## 错误处理

- 如果学校代码已存在，会提示"学校代码已存在"
- 如果边界点少于 3 个，会提示"至少需要3个点才能构成多边形边界"
- 如果坐标格式错误，会提示相应的错误信息
