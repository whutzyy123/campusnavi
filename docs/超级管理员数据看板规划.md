# 超级管理员数据看板详细规划

**实现状态**：已实现（2026-03）

---

## 一、规划原则

1. **核心指标**：用户增长、留存、核心功能使用率
2. **趋势优先**：折线图展示时间序列，便于发现拐点与异常
3. **结构洞察**：饼图/柱状图展示构成，便于优化资源分配
4. **可对比**：支持环比/同比思维，便于归因分析
5. **可执行**：数据与 actionable 决策关联（如沉默用户需召回）

---

## 二、各模块详细规划

### 1. 用户增长详细页 `/super-admin/analytics/users`

| 图表 | 类型 | 数据维度 | 产品价值 |
|------|------|----------|----------|
| 新增用户趋势 | 折线图 | 按日/周的新增用户数（最近 30 天） | 观察增长曲线、活动/推广效果 |
| 累计用户趋势 | 折线图 | 每日累计注册用户数 | 观察整体规模增长 |
| 新增用户按学校分布 | 柱状图 | 各学校最近 30 天新增用户数 | 识别高增长学校，优化投放 |
| 活跃学校用户数 | 柱状图 | 各学校当前用户数 | 学校规模对比 |

**时间粒度**：支持近 7 天 / 近 30 天切换

## 三、技术实现要点

1. **图表库**：Recharts（轻量、React 友好、支持折线/柱状/饼图）
2. **数据接口**：`lib/admin-analytics-actions.ts` 统一按 `days`（7/30/365）返回时序数据
3. **组件复用**：`ChartCard`、`ChartLoadingState`、`ChartEmptyState`、`AnalyticsGranularityTabs`
4. **路由结构**：`/super-admin/analytics` 重定向至 users；`/super-admin/analytics/users` 等为子页
5. **日期工具**：`lib/analytics-utils.ts` 提供 `formatChartDate`、`tooltipLabelFormatter`

---

## 四、导航入口

- 系统看板首页：各 StatCard 增加「查看详情」链接，跳转至对应数据分析子页
- 侧边栏：新增「数据分析」菜单，展开为各子页
- 导出报表：看板与数据分析页均有「导出报表」按钮，支持周报/月报/年报 CSV 下载

---

## 五、周报 / 月报 / 年报导出

| 类型 | 周期 | 说明 |
|------|------|------|
| 周报 | 近 7 天 | 过去 7 天汇总 |
| 月报 | 近 30 天 | 过去 30 天汇总 |
| 年报 | 近 365 天 | 过去一年汇总 |

**导出内容**：用户增长、用户留存、生存集市、地图与内容、核心率指标、消息。CSV 格式，UTF-8 BOM 编码，便于 Excel 打开。

---

## 六、核心率指标（有效数据）

| 率指标 | 计算公式 | 分母条件 |
|--------|----------|----------|
| 用户活跃率 | MAU / 总用户数 | 总用户 > 0 |
| 集市成交率 | 成交 / (成交 + 过期) | 已结束商品 > 0 |
| 集市过期率 | 过期 / (成交 + 过期) | 已结束商品 > 0 |
| 留言互动率 | 有点赞留言 / 总留言 | 总留言 > 0 |
| 失物招领完成率 | 已找到 / (已找到 + 已过期) | 已结束事件 > 0 |
| 反馈处理率 | 已处理 / 总反馈 | 总反馈 > 0 |
| 留言举报处理率 | 已审核 / 有举报留言 | 有举报留言 > 0 |

分母为 0 时，对应率显示为 `—`。
