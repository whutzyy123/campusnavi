## 四、架构与约定

### 4.1 目录结构

```
src/
├── app/              # Next.js App Router（页面、布局、API）
├── components/      # 可复用组件
├── lib/              # 工具、Server Actions、业务逻辑
├── store/            # Zustand 状态
├── hooks/            # 自定义 Hooks
├── prisma/           # Schema、迁移、种子
└── scripts/          # 迁移与工具脚本
```

### 4.2 数据流

| 场景 | 方式 | 说明 |
|------|------|------|
| 服务端初始数据 | RSC / Server Components | 首屏直出 |
| 客户端数据变更 | Server Actions | 表单提交、按钮操作 |
| 客户端数据获取 | Route Handlers (`/api/*`) | fetch 或兼容 |
| 全局 UI 状态 | Zustand | 认证、学校、导航、筛选 |

### 4.3 认证

- **方式**：HTTP Only Cookie（`campus-survival-auth-token`）
- **存储**：服务端 Set-Cookie，客户端不可读
- **鉴权**：`getAuthCookie()` 读取，`requireAdmin` / `requireSuperAdmin` 校验

### 4.4 多租户

- **标识**：`schoolId` 贯穿业务表
- **查询**：`where: { schoolId: currentSchoolId }`（超管除外）
- **写入**：`schoolId` 从会话注入，不信任客户端
